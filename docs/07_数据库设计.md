# 数据库设计

---

# 1. 文档目标

本文档定义系统持久化层设计规范。

系统采用：

```text
PostgreSQL
+
Qdrant
```

双存储架构。

职责划分：

```text
PostgreSQL
负责结构化业务数据

Qdrant
负责语义检索与长期记忆
```

---

# 2. 设计原则

## 原则一：职责分离

PostgreSQL：

```text
事务数据
业务数据
运行记录
```

---

Qdrant：

```text
Embedding
语义检索
长期记忆
```

---

禁止：

```text
Qdrant保存业务状态

PostgreSQL承担向量检索
```

---

## 原则二：Repository模式

业务层禁止直接访问数据库。

禁止：

```python
session.execute(...)
```

出现在：

```text
Agent

Service

Workflow
```

中。

---

必须通过：

```python
TaskRepository

ReportRepository

SourceRepository
```

访问。

---

## 原则三：迁移管理

所有表结构变更必须通过：

```text
Alembic
```

管理。

禁止：

```sql
ALTER TABLE ...
```

手工修改数据库。

---

## 原则四：可追溯

所有研究结果必须能够追溯：

```text
来源

时间

任务

工作流
```

---

# 6. PostgreSQL表设计

---

## tasks

研究任务表。

---

用途：

记录用户发起的研究任务。

---

字段：

```python
id: UUID

task_name: str

user_query: str

task_type: str

status: str

created_at: datetime

updated_at: datetime
```

---

说明：

```text
对应AgentState.task_id
```

---

索引：

```sql
idx_tasks_status

idx_tasks_created_at
```

---

# 7. workflow_runs

工作流执行记录。

---

用途：

记录每一次Workflow执行。

---

一个Task代表一个研究主题。

一个Task允许关联多个WorkflowRun。

用于支持：

- 手动重跑
- 定时更新
- 模型升级后的重新分析
- 失败任务重试

例如：

```text
Task:
MCP发展趋势研究

WorkflowRun #1
2026-08-01

WorkflowRun #2
2026-08-02

WorkflowRun #3
2026-08-10
```

---

字段：

```python
id: UUID

task_id: UUID

run_number: int

trigger_type: str

workflow_version: str

status: str

started_at: datetime

finished_at: datetime

duration_ms: int

error_message: str | None
```

---

trigger_type：

```text
manual    # 手动触发
schedule  # 定时触发
retry     # 失败重试
api       # API触发
```

---

关系：

```text
Task
1
↓

N

WorkflowRun
```

---

索引：

```sql
idx_workflow_runs_task_id

idx_workflow_runs_status
```

---

# 8. sources

采集结果元数据表。

---

用途：

---

```sql
idx_sources_collected_at
```

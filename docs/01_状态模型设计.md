# 状态模型设计

---

# 1. 文档目标

本文档定义系统唯一状态模型（State Model）。

本项目采用：

- LangGraph 作为工作流引擎
- AgentState 作为唯一状态载体
- Pydantic 作为状态定义框架

系统中所有 Agent：

输入：

```python
AgentState
```

输出：

```python
AgentState
```

禁止传递其他业务对象。

---

# 2. 状态设计原则

## 原则一：单一状态源

系统运行期间：

```python
AgentState
```

是唯一事实来源（Single Source of Truth）。

---

## 原则二：模块化状态

状态按照职责划分：

```text
Research
Context
Analysis
Memory
Report
```

每个模块独立维护。

---

## 原则三：可序列化

所有状态必须：

- Pydantic定义
- JSON可序列化
- 支持Checkpoint持久化

---

## 原则四：支持工作流恢复

未来支持：

```text
断点恢复

人工介入

工作流回放
```

因此所有关键状态必须持久化。

---

class AgentState(BaseModel):

# 6. ResearchState

负责研究任务管理。

```python
class ResearchState(BaseModel):

    topic: str = ""

    keywords: list[str] = []

    data_sources: list[str] = []

    search_round: int = 0

    need_more_data: bool = False

    information_gaps: list[str] = []

    papers: list[PaperRecord] = []

    repositories: list[RepositoryRecord] = []

    models: list[ModelRecord] = []
```

---

## 字段说明

### topic

研究主题。

例如：

```text
MCP
```

---

### keywords

研究关键词。

例如：

```python
[
    "MCP",
    "Model Context Protocol"
]
```

---

### search_round

当前研究轮次。

用于控制：

```text
Deep Research Loop
```

---

### information_gaps

分析阶段发现的信息缺口。

例如：

```python
[
    "缺少GitHub生态数据"
]
```

# 10. MemoryState

负责记忆系统状态。

---

```python
class MemoryState(BaseModel):

    memory_ids: list[str] = []

    memory_updated: bool = False
```

---

说明：

MemoryState 不保存长期记忆内容。

长期记忆保存在：

```text
Qdrant
```

MemoryState 仅记录：

```text
本次Workflow的记忆操作结果
```

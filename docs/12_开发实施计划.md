# 开发实施计划

---

# 1. 文档目标

本文档定义项目开发实施路线。

目标：

```text
让 Claude Code 能够按照固定顺序逐步实现系统。

避免：

一次生成整个项目
模块耦合失控
后期返工
```

---

开发原则：

```text
MVP优先

功能切片优先

每个阶段均可运行

每个阶段均可验收

每个阶段均可演示
```

---

# 2. 开发策略

采用：

```text
Milestone 驱动开发
```

而非：

```text
技术层驱动开发
```

---

不采用：

```text
数据库
↓
Repository
↓
Service
↓
API
↓
前端
```

方式。

---

采用：

```text
功能闭环
↓
逐步扩展
```

方式。

---

每个 Milestone 必须满足：

```text
可运行

可验证

可演示

可提交
```

---

# 12. Claude Code 开发规范

---

## 原则一

一次只开发：

```text
一个 Milestone
```

---

禁止：

```text
同时开发多个Milestone
```

---

## 原则二

完成当前Milestone后：

必须进行：

```text
代码审查

运行验证

功能验收
```

---

再进入下一阶段。

---

## 原则三

开发顺序严格遵守：

```text
Milestone 1
↓
Milestone 2
↓
Milestone 3
↓
...
```

---

禁止跳跃开发。

---

# 13. Claude Code Prompt规范

必须明确：

```text
当前Milestone
```

示例：

```text
当前进入 Milestone 3

目标：

涉及模块：

验收标准：
```

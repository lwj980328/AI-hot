# 工具设计规范

---

# 1. 文档目标

本文档定义系统工具层（Tool Layer）的统一规范。

工具层负责：

- 外部数据获取
- 第三方服务访问
- MCP服务访问
- 数据预处理

Agent不得直接访问外部资源。

所有外部能力必须通过 Tool 层提供。

---

# 2. 设计原则

## 原则一：Agent只依赖抽象

Agent不得直接依赖：

- GithubTool
- ArxivTool
- HuggingFaceTool

Agent只能通过 ToolRegistry 获取工具。

---

## 原则二：工具统一接口

所有工具必须继承：

```python
BaseTool
```

统一生命周期。

统一调用方式。

统一异常处理。

---

## 原则三：本地工具与MCP工具统一抽象

未来系统支持：

- Local Tool
- MCP Tool

Agent无需感知差异。

---

## 原则四：工具可独立测试

每个工具必须支持：

- 单元测试
- Mock测试
- 集成测试

---

# 3. 工具架构

整体结构：

```text
Agent
  │
  ▼

  ├── HuggingFaceTool
```

Agent只与：

```text
ToolRegistry
```

交互。

# 4. BaseTool抽象基类

所有工具必须继承：

```python
BaseTool
```

---

## 标准定义

```python
from abc import ABC
from abc import abstractmethod

class BaseTool(ABC):

    name: str

    description: str

    version: str

    @abstractmethod
    async def execute(
        self,
        input_data: BaseModel
    ) -> BaseModel:
        pass
```

---

## 字段说明

### name

工具唯一名称。

例如：

```python
"github_search"
```

---

### description

工具描述。

用于Agent理解工具用途。

---

### version

工具版本号。

用于后续升级管理。

---

### execute()

统一执行入口。

所有工具必须实现。

---

# 5. ToolInput规范

所有工具必须定义：

```python
InputSchema
```

---

示例：

```python
class GithubSearchInput(BaseModel):

    limit: int = 20
```

---

禁止：

```python
dict
```

作为输入。

---

示例：


---

禁止：

```python
dict
```

作为输出。

# 10. ToolRegistry

## 设计目标

Agent只能通过注册表获取工具。

禁止：

```python
GithubTool()
```

直接实例化。

---

正确方式：

```python
tool = registry.get_tool(
    "github_search"
)
```

---

# 11. ToolRegistry定义

```python
class ToolRegistry:

    def register(
        self,
        tool: BaseTool
    ):
        pass

    def get_tool(
        self,
        name: str
    ) -> BaseTool:
        pass
```

---

## 功能

支持：

### 工具注册

```python
register(tool)
```

---

### 工具发现

```python
get_tool(name)
```

---

### 工具列表

```python
list_tools()
```

---

### 工具版本查询

```python
get_version()
```

---

# 12. Agent工具调用规范

禁止：

```python
tool = GithubTool()
```

---

禁止：

```python
from tools.github_tool import GithubTool
```

直接在Agent中使用。

---

必须：

```python
tool = registry.get_tool(
    "github_search"
)
```

---

执行：

```python
result = await tool.execute(
    input_data
)
```

# 17. 后续扩展原则

新增工具时：

不得修改Agent。

不得修改Workflow。

不得修改State。

只允许：

1. 实现BaseTool

2. 注册ToolRegistry

即可接入系统。
